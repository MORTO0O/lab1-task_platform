# Лабораторная работа №2
**Модель задачи: дескрипторы и @property**

**Тема:** Инкапсуляция, дескрипторы (Data/Non-data), свойства и обработка исключений.

## Цель работы
Перейти от простых структур данных к защищенной доменной модели. Реализовать валидацию атрибутов через протокол дескрипторов и управление доступом через `@property`.

## Новые возможности
- **Валидация через дескрипторы**: Поле `priority` автоматически проверяет тип данных и вхождение в диапазон [1, 5].
- **Инкапсуляция**: Идентификатор задачи `id` доступен только для чтения.
- **Управление состоянием**: Сеттер `status` контролирует допустимые переходы и значения.
- **Логирование доступа**: Non-data дескриптор `access_log` фиксирует обращения к объекту.
- **Кастомные исключения**: Иерархия ошибок в `exceptions.py` для точной диагностики.
