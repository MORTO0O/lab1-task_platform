# Лабораторная работа №3
**Очередь задач: итераторы и генераторы**

## Цель работы
Научиться реализовывать пользовательские коллекции и ленивую (потоковую) обработку данных. Разработать очередь задач (`TaskQueue`), поддерживающую итерацию и эффективную фильтрацию без избыточного расхода памяти.

## Новые возможности
- **Пользовательская коллекция**: Реализован класс `TaskQueue` для хранения и управления списком задач.
- **Протокол итерации**: Очередь полностью совместима со стандартными конструкциями Python. Задачи можно перебирать в цикле `for`, а также передавать очередь во встроенные функции вроде `list()` благодаря реализации магического метода `__iter__`.
- **Многократный обход**: Очередь спроектирована так, что её можно итерировать неограниченное количество раз без потери данных.
- **Ленивые фильтры**: Реализована фильтрация задач (по статусу и приоритету) с использованием генераторов (ключевое слово `yield`). Это позволяет извлекать нужные задачи "на лету", не создавая копий списков и не расходуя лишнюю оперативную память.

## Как запустить тесты

```powershell
python -m pytest src/tests --cov=src
```


### Чему научился

Интегрировать собственные классы в стандартные механизмы Python (протокол итерации).
Использовать ключевое слово yield для создания генераторных функций.
Понимать принцип "ленивых вычислений" (lazy evaluation) и применять его для экономии ресурсов компьютера при работе с коллекциями.
Избегать создания промежуточных списков в памяти при фильтрации и поиске данных.
